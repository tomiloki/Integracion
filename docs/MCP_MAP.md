# Mapa MCP y herramientas sugeridas

## GitHub
Uso recomendado:
- Revisar PRs, issues, comentarios, estado de checks.
- Gestionar feedback por etapas pequenas.

## Browser automation (Playwright/Agent Browser)
Uso recomendado:
- Verificar flujos UI completos:
  - login
  - compra
  - checkout webpay callback
  - backoffice admin

## Vercel / Deploy tools
Uso recomendado:
- Preview deployments para validar UX y feedback rapido.
- Logs de runtime para diagnostico.

## Docs tools
Uso recomendado:
- Mantener referencia actualizada de APIs/SDKs.
- Evitar decisiones sobre memoria para integraciones sensibles.

## Regla practica
Elegir siempre la herramienta que minimice riesgo y maximice evidencia reproducible.
